About Luga JS

Luga JS encompass a set of data-centric JavaScript libraries, specialized tools that do few things and try to do them well. Luga JS is not intended and will never be a framework, we believe in keeping our tools small, focused and sharp.

Luga JS requires jQuery, this doesn't means we believe jQuery is the best JavaScript framework available. We just feel itâ€™s better to use a generic framework for things like XHR, DOM traversing and manipulation, and jQuery is very popular, lightweight and well-know.

Right now the following production ready libraries are available:

  • Validator: a library for client-side form validation
  • CSI: s simple solution to perform includes client-side

Each libraries requires luga.core.js and is available as separated .js file (either plain or minified). This way you can use them separately, depending on your own needs.

As an alternative you can use the concatenated file that contains all libraries and is also available in minified format

The project is hosted on GitHub

Unit test

Luga JS contains a comprehensive set of unit tests, developed with Jasmine

The unit tests play a fundamental role in ensuring the code is robust and, at the very same time, provide plenty of useful information about the API
