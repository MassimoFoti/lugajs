Why regions are rendered so often?

Region associated with a DataSet may indeed be rendered more often than you would have expected. This happens because certain events may change both the data contained inside the DataSet and its state. Sometimes instead only the data is changed, like after filtering or sorting, without affecting the state at all.

Since regions are designed to reflect both the data and the DataSet's state, they may react to changes that happens in quick succession.

There is no need to worry about this, even on very large amount of data the user experience is not affected.
