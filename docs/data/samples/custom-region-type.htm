Custom regions and templating

You can fully customize the way a region is rendered, either using a different templating engine or not using templating at all.

Add the data-lugads-regiontype attribute with a value that matches an existing JavaScript constructor function:

  1. <div data-lugads-region="detailsRegion" data-lugads-datasource="lady" data-lugads-template="detailsTemplate" data-lugads-regiontype="underscoreTemplate></div>

The constructor should extend the luga.data.region.Base class and override its render() method:

  1. /**
  2. * This simplified implementation of a custom region handler should not be considered production-ready
  3. * @extends luga.data.region.Base
  4. */
  5. var underscoreTemplate = function(options){
  6. luga.extend(luga.data.region.Base, this, [options]);
  7. // Underscore specific part
  8. var templateId = this.config.node.attr(luga.data.region.CONST.CUSTOM_ATTRIBUTES.TEMPLATE);
  9. var templateStr = jQuery("#" + templateId).html();
  10. var compiledTemplate = _.template(templateStr);
  11. /** @override */
  12. this.render = function(){
  13. var records = this.dataSource.getContext().context;
  14. if(records !== null){
  15. this.config.node.html(compiledTemplate(records));
  16. }
  17. };
  18. };

For default rendering, using Handlebars as template engine, there is no need to specify the data-lugads-regiontype attribute, since luga.data.Region is used by default.
