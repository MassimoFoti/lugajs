Syntax reference

The complete syntax reference for HTML5 custom data attributes is included below

Form tag
data-lugavalidator-validate Required. Set it to true in order to enable validation on the current form
data-lugavalidator-error

Optional. Function that will be invoked to handle/display validation messages

Default to luga.validator.errorAlert (display plain alert messages)

A more sophisticated handler is also available and you can write your own too
data-lugavalidator-before Optional. Function that will be invoked before validation is performed. See sample
data-lugavalidator-after Optional. Function that will be invoked after the form is successfully validated. See sample
data-lugavalidator-blocksubmit

Optional (default to true). By default submit buttons are disabled after the form is submitted. This prevents multiple submits but also prevents the value of the submit buttons from being passed as part of the HTTP request. Set this attribute to false to keep the submit buttons enabled

If you expect your users to be able to use the "back" button and resubmit the form, you will need to set data-lugavalidator-blocksubmit="false" or take other steps to re-enable the submit buttons when returning to the form

Text fields and textarea tags
data-lugavalidator-required

The single most important attributes, set it to true to flag the field as required

In case you need conditional validation, set it to the name of a custom function
data-lugavalidator-message Optional. The error message associated with the field
data-lugavalidator-errorclass Optional. CSS class that will be applied to the field if invalid
data-lugavalidator-pattern

Optional. Validation pattern to be applied, either built-in or custom. There are 9 generic built-in patterns:

  • lettersonly
  • alphanumeric (see sample)
  • integer
  • positiveinteger
  • number
  • filepath_pdf
  • filepath_jpg
  • filepath_zip
  • filepath
  • time HH:MM

Custom patterns can be added too
data-lugavalidator-minlength Optional. Enforce a minimum text length
data-lugavalidator-maxlength Optional. Enforce a maximum text length
data-lugavalidator-minnumber Optional. Enforce a minimum numeric value
data-lugavalidator-maxnumber Optional. Enforce a maximum numeric value
data-lugavalidator-datepattern

Optional. Date format pattern to be applied, either built-in or custom. There are 14 built-in date patterns:

  • YYYY-MM-DD
  • YYYY-M-D
  • MM.DD.YYYY
  • M.D.YYYY
  • MM/DD/YYYY
  • M/D/YYYY
  • MM-DD-YYYY
  • M-D-YYYY
  • DD.MM.YYYY
  • D.M.YYYY
  • DD/MM/YYYY
  • D/M/YYYY
  • DD-MM-YYYY
  • D-M-YYYY

Custom date patterns can be added too
data-lugavalidator-mindate

Optional. Enforce a minimum date

If no data-lugavalidator-datepattern is specified, default pattern (YYYY-MM-DD) will be used to parse and compare dates
data-lugavalidator-maxdate

Optional. Enforce a maximum date

If no data-lugavalidator-datepattern is specified, default pattern (YYYY-MM-DD) will be used to parse and compare dates
data-lugavalidator-equalto

Optional. Id of another field who's values will be compared for equality (see sample)
Text fields can also trigger custom validation
Checkboxes
data-lugavalidator-minchecked Specify a minimum number of boxes that can be checked in a group. Set it to 1 to allow only one choice (see sample)
data-lugavalidator-maxchecked Specify a maximum number of boxes that can be checked within a group (see sample)
data-lugavalidator-message Optional. The error message associated with the field
data-lugavalidator-errorclass Optional. CSS class that will be applied to the field if invalid
Radio buttons
data-lugavalidator-required Set it to true to flag group of buttons as required (see sample)
data-lugavalidator-message Optional. The error message associated with the field
data-lugavalidator-errorclass Optional. CSS class that will be applied to the field if invalid
Select tags
data-lugavalidator-invalidindex Optional. Prevents selection of an entry on a given position, zero based (see sample)
data-lugavalidator-invalidvalue Optional. Prevents selection of an entry with a given value (see sample)
data-lugavalidator-message Optional. The error message associated with the field
data-lugavalidator-errorclass Optional. CSS class that will be applied to the field if invalid
Submit buttons
data-lugavalidator-disabledlabel

Optional. Display this string as the button's label during form processing, after successful validation

Large forms, especially the ones with file upload, can require some processing time, during this process it's better to avoid multiple submits from the user. That's why the validator automatically disables all the submit buttons after a form successfully passes the validation process
