Checkboxes

Checkboxes are a special kind of field, since they often come in groups, sharing the same name but having different values.

Luga Validator validates a set of checkboxes with the same name as a unique entity. So, inside a set of checkboxes, you can define the validation attributes only once (if you specify attributes with conflicting values, the last one wins):

  1. <input type="checkbox" name="drink" data-luga-minchecked="2" value="Nicole" />
  2. <input type="checkbox" name="drink" value="Kate" />

There are two different attributes for checkboxes validation. data-luga-minchecked ensures that at least a minimum number of boxes are checked in a group:

  1. <input type="checkbox" name="drink" data-luga-minchecked="2" value="Nicole" />

luga-data-maxchecked specifies a maximum number of boxes that can be checked in a group:

  1. <input type="checkbox" name="drink" luga-data-maxchecked="3" value="Nicole" />

You can use both attributes at the same time:

  1. <input type="checkbox" name="drink" luga-data-minchecked="2" luga-data-maxchecked="3" value="Nicole" />

If we want only one box checked, set both attributes to 1:

  1. <input type="checkbox" name="pasta" luga-data-minchecked="1" luga-data-maxchecked="1" value="Penne" />
