Custom patterns

Luga Validator was designed with extensibility in mind. That's why all the validation rules and patterns are stored inside a JavaScript object.

For this custom validation you want to be sure the user enters a string starting with http://. Our regular expression is ^http://

Luga Validator stores all patterns inside luga.validator.patterns, a JavaScript object, acting as a container for both built-in and custom patterns.

So all you need to do is to add the following line of JavaScript code: 

luga.validator.patterns.httpstart = new RegExp("^http://");

The HTML will look like: 

<input type="text" name="theurl"
	data-lugavalidator-required="true"
	data-lugavalidator-pattern="httpstart"
	data-lugavalidator-message="Please URL!"/>
