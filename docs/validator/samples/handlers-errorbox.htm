Use luga.validationHandlers.errorBox as error handler

By default the validator uses JavaScript alert boxes to display errors to the user. If we want to display errors in a more user friendly way, use "luga.validationHandlers.errorBox" as error handler

If you need more flexibility, you can create a custom handler.

Styling and formatting for error box are specified inside an external CSS file. You can tweak and modify it to suit your needs.

  1. <form data-luga-validate="true" data-luga-error="luga.validationHandlers.errorBox">

Override default error handler

The value for the default error handler is stored inside luga.validator.CONST.HANDLERS.FORM_ERROR. You can override it with something that fits your needs

  1. <script>
  2. luga.validator.CONST.HANDLERS.FORM_ERROR = "luga.validationHandlers.errorBox";
  3. </script>
