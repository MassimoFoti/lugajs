Error handler

By default Ajaxform uses luga.ajaxform.handlers.errorAlert as error handler. It displays either a standard message or the content of the data-lugajax-errormsg custom attribute inside an alert 

<form action="missing.json" data-lugajax-form="true" data-lugajax-errormsg="Your
	error message here">
	<input type="text" name="name" />
	<input type="submit" value="Submit"/>
</form>
Default error handler
luga.ajaxform.handlers.errorBox

You can use the built-in luga.ajaxform.handlers.errorBox function instead. It works better than a crude alert

Just set the data-lugajax-error custom attribute 

<form action="missing.json" data-lugajax-form="true"
	data-lugajax-error="luga.ajaxform.handlers.errorBox">
	<input type="text" name="name"/>
	<input type="submit" value="Submit"/>
</form>
errorBox
Custom error handler

You can write your own error handler and use it instead 

handlers = {};
	handlers.alertHttp = function(msg, formNode, response){
	// Alert the HTTP status and error message in a error box above the form
	alert(response.status + " " + response.statusText);;
};
<form action="missing.json" data-lugajax-form="true"
	data-lugajax-error="handlers.alertHttp">
	<input type="text" name="name"/>
	<input type="submit" value="Submit"/>
</form>

Handler's signature
The handler will be invoked with 3 arguments
Arguments
msg {String} Value of the data-lugajax-errormsg attribute
formNode {jQuery} jQuery object wrapping the form
response {luga.xhr.response} Luga JS wrapper around XMLHttpRequest
