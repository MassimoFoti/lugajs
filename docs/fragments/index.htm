About Luga JS

Luga JS encompass a set of JavaScript libraries, designed to build form and data centric applications, specialized tools that do few things and try to do them well. Luga JS is not intended and will never be a framework, we believe in keeping our tools simple, atomic yet flexible.

Luga JS is designed to work in IE11+ and most recent versions of Chrome, FireFox and Safari

The following production ready libraries are available:

  • Common: a minimal set of shared API and utilities
  • Luga Data: data, binding and dynamically generated HTML
  • Validator: client-side form validation

Other libraries are available as separated projects:

  • Luga Router: a feature-rich JavaScript library for client-side routing. It's compact, focused and extensible
Dependencies

Until version 0.9.7 used to require jQuery in order to stay compatible with older browsers. At that point we realized it was time for a change, we gave up supporting IE9 and IE10 and managed to get rid of the dependency. Unfortunately this came at a cost, since the signature of various API changed in a way that is not fully backward compatible

At the moment the documentation is still using Bootstrap and jQuery, but this is only for convenience, the codebase does not require any of these two

Handlebars
By default Luga Data regions require Handlebars, but you can customize the way regions are rendered, without using Handlebars
Bower

You can install Luga JS using bower by running: bower install lugajs

Unit tests

Luga JS contains a comprehensive set of unit tests, developed with Jasmine

The unit tests play a fundamental role in ensuring the code is robust and, at the very same time, provide plenty of useful information about the API

Code coverage is usually around 100%. You can check the coverage report by yourself
