About Luga JS

Luga JS encompass a set of JavaScript libraries, designed to build form and data centric applications, specialized tools that do few things and try to do them well. Luga JS is not intended and will never be a framework, we believe in keeping our tools simple, atomic yet flexible.

Luga JS currently requires jQuery, this doesn't means we believe jQuery is the best JavaScript framework available. We just feel itâ€™s convenient to use it for things like DOM traversing and manipulation. Future versions will drop support for older browsers and most likely will not require jQuery anymore

Right now the following production ready libraries are available:

  • Common: a minimal set of shared API and utilities
  • Luga Data: data, binding and dynamically generated HTML
  • Validator: client-side form validation
Unit tests

Luga JS contains a comprehensive set of unit tests, developed with Jasmine

The unit tests play a fundamental role in ensuring the code is robust and, at the very same time, provide plenty of useful information about the API

Code coverage is usually around 100%. You can check coverage report by yourself

Browser and jQuery compatibility

Luga JS is designed to work in IE10+ and most recent versions of Chrome and FireFox

The codebase is developed and tested alongside jQuery 3.x. We occasionally test it alongside version 2.x to ensure it's fully forward compatible and it should work just fine on that version too

If you use jQuery version 1.x instead, you might run without major issues in older browser versions, but please be aware we do not actively test or support this configuration

Bower

You can install Luga JS using bower by running: bower install lugajs
