About Luga JS

Luga JS encompass a set of JavaScript libraries, designed to build form and data centric applications, specialized tools that do few things and try to do them well. Luga JS is not intended and will never be a framework, we believe in keeping our tools small, focused and sharp.

Luga JS requires jQuery, this doesn't means we believe jQuery is the best JavaScript framework available. We just feel itâ€™s better to use a generic framework for things like XHR, DOM traversing and manipulation, jQuery is very popular, lightweight and well-know.

Right now the following production ready libraries are available:

  • Ajaxform: turn any HTML <form> into a form that get submitted by XHR, without the need to reload the page
  • Luga Data: data, binding and dynamically generated HTML
  • Validator: client-side form validation
Unit tests

Luga JS contains a comprehensive set of unit tests, developed with Jasmine

The unit tests play a fundamental role in ensuring the code is robust and, at the very same time, provide plenty of useful information about the API

Code coverage is usually around 100%. You can check coverage report by yourself

Browser and jQuery compatibility

Luga JS is designed to work in IE9+ and most recent versions of Chrome and FireFox

The codebase is developed and tested alongside jQuery 2.x. We occasionally test it alongside version 3.x to ensure it's fully forward compatible and it should work just fine on that version too

If you use jQuery version 1.x instead, you might run without major issues in older browser versions, but please be aware we do not actively test or support this configuration
